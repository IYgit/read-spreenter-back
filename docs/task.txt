Ми розробляємо Web додаток для розвитку навиків швидкочитання.
Наразі частково розроблений MVP фронтенда (папка speed-reader-front).
Фронтенд дозволяє користувачу вибрати вправу, виконати її, побачити результат та статистику виконання вправи.


Твоя задача:
1. вивчити фронтенд
2. запропонувати функціонал бекенда.
3. планується використовуати БД PostgreSQL та Flyway.
4. Передбачається, що додаток буде надавати можливість працювати в двох режимах:
     - одиночний (користувач виконує вправу самостійно)
     - груповий режим (2 користувача виконують вправу разом) і порівнюють свої результати.


Груповий режим передбачає створення мультиплейєра, в якому екран ділиться навпіл для 2 користувачів.
Обидва користувачі в режимі реального часу мають можливість спостеріагти прогрес один одного та порівнювати свої результати.
Перший користувач виконує вправу з лівої сторони, другий - з правої.
Цей режим передбачає синхронізацію екранів користувачів, яку повинен забезпечити бекенд.

Опиши в md файлі функціонал бекенда, який слід реалізувати для цього додатка.
Втому числі, опиши функціонал, який наразі на фронтенді не реалізований:
    - можливість вибору режиму (одиночний чи груповий);
    - можливість вибору користувача для групового режиму;
    - синхронізація екранів користувачів у груповому режимі;
    - збереження результатів та статистики виконання вправ для кожного користувача.

Якщо я не передбачив якісь функції, які можуть бути корисними для цього додатка, будь ласка, запропонуй їх.

Запропонуй можливі рішення в md файлі.
